# Kinoman - Бэкенд онлайн-кинотеатра

> [English version](./README.md)

Production-ready REST API бэкенд для полнофункциональной платформы онлайн-кинотеатра. Построен на **NestJS** и **MongoDB** и обеспечивает работу сервиса стриминга фильмов: аутентификация пользователей, ролевой контроль доступа, управление каталогом фильмов, система рейтингов, избранное и Telegram-уведомления о новых релизах.

## Обзор проекта

**Kinoman Backend** - серверная часть приложения онлайн-кинотеатра. Предоставляет RESTful API для фронтенд-клиента: регистрация и вход пользователей через JWT, управление каталогом фильмов, система оценок, загрузка файлов и Telegram-бот для уведомлений о новых фильмах.

---

## Стек технологий

| Категория | Технология |
|---|---|
| Фреймворк | NestJS 10 + TypeScript 5 |
| База данных | MongoDB (через Mongoose + Typegoose) |
| Аутентификация | Passport.js + JWT (access + refresh токены) |
| Хэширование паролей | bcryptjs (10 раундов соли) |
| Загрузка файлов | Multer + fs-extra + NestJS Serve Static |
| Telegram-бот | Telegraf 4 |
| Валидация | class-validator + class-transformer |
| Тестирование | Jest + Supertest |
| Линтинг / Форматирование | ESLint + Prettier |

## Архитектура

Проект следует модульной, feature-based архитектуре NestJS с чётким разделением ответственности:

```
src/
├── auth/          # Регистрация, вход, обновление токена
├── user/          # Профиль, избранное, CRUD пользователей (admin)
├── movie/         # Каталог фильмов, счётчик просмотров, поиск
├── genre/         # Управление жанрами, коллекции
├── actor/         # Управление актёрами с подсчётом фильмов
├── rating/        # Рейтинги фильмов от пользователей
├── file/          # Загрузка файлов и раздача статики
├── telegram/      # Уведомления через Telegram-бот
├── config/        # Конфигурация MongoDB и JWT
├── shared/        # Перечисления, константы
├── pipes/         # IdValidationPipe (валидация ObjectId)
└── main.ts        # Bootstrap: порт 4200, префикс /api, CORS
```

**Применяемые паттерны:**
- MVC на каждый модуль (Controller → Service → Model)
- Dependency Injection через IoC-контейнер NestJS
- Guard-авторизация (`JwtAuthGuard`, `OnlyAdminGuard`)
- Кастомные декораторы: `@Auth()`, `@User()`
- Валидация DTO через class-validator
- MongoDB aggregation pipeline (актёры с количеством фильмов)
- Upsert-паттерн для рейтингов (создать или обновить)
- Mongoose `.populate()` для загрузки связанных документов

## Схема базы данных

### User (Пользователь)
```ts
{
  email: string         // уникальный
  password: string      // bcrypt-хэш
  isAdmin: boolean      // по умолчанию: false
  favorites: ObjectId[] // ссылки на Movie
}
```

### Movie (Фильм)
```ts
{
  poster: string
  bigPoster: string
  title: string
  slug: string          // уникальный, URL-friendly идентификатор
  videoUrl: string
  parameters: {
    year: number
    duration: number    // в минутах
    country: string
  }
  rating: number        // по умолчанию: 4.0, пересчитывается при оценках
  countOpened: number   // счётчик просмотров
  genres: ObjectId[]    // ссылки на Genre
  actors: ObjectId[]    // ссылки на Actor
  isSendTelegram: boolean // отправлено ли уведомление в Telegram
}
```

### Genre (Жанр)
```ts
{
  name: string
  slug: string        // уникальный
  description: string
  icon: string        // название иконки или URL
}
```

### Actor (Актёр)
```ts
{
  name: string
  slug: string  // уникальный
  photo: string
}
```

### Rating (Рейтинг)
```ts
{
  userId: ObjectId    // ссылка на User
  movieId: ObjectId   // ссылка на Movie
  value: number       // оценка пользователя
}
```

## Аутентификация и авторизация

**Процесс:**
1. Пользователь регистрируется с email + паролем → пароль хэшируется bcryptjs (10 раундов соли)
2. При входе возвращается **пара токенов**: короткоживущий access-токен (1 ч) + долгоживущий refresh-токен (10 дней)
3. Клиент отправляет `Authorization: Bearer <accessToken>` в защищённых запросах
4. `JwtAuthGuard` проверяет токен через стратегию Passport JWT
5. `OnlyAdminGuard` проверяет флаг `user.isAdmin` для admin-эндпоинтов

**Кастомные декораторы:**
- `@Auth(Roles.User)` - требует аутентифицированного пользователя
- `@Auth(Roles.Admin)` - требует роль администратора
- `@User('email')` - извлекает конкретное поле из JWT payload

## Ключевые возможности

- **JWT-аутентификация** - пара токенов с раздельными сроками жизни
- **Ролевой контроль доступа** - guard'ы user/admin на всех чувствительных эндпоинтах
- **Каталог фильмов** - полный CRUD, slug-URL, поиск, счётчик просмотров
- **Жанровые коллекции** - жанры возвращаются с обложками представительных фильмов
- **Агрегация актёров** - список актёров с количеством фильмов через MongoDB pipeline
- **Система рейтингов** - оценки от пользователей с автоматическим пересчётом среднего
- **Избранное** - добавление/удаление фильмов из персонального списка
- **Загрузка файлов** - файлы организованы по папкам, раздаются как статика через `/uploads`
- **Telegram-уведомления** - бот отправляет постер и ссылку в канал при публикации нового фильма (только в продакшене)
- **Поиск** - регистронезависимый regex-поиск по фильмам, жанрам, актёрам и пользователям

## Быстрый старт

### Требования

- Node.js ≥ 20.3.1
- MongoDB (локально или Atlas)
- npm или yarn

### Установка

```bash
git clone <repository-url>
cd Full-online-cinema-be
npm install
```

### Настройка окружения

Создайте файл `.env` в корне проекта:

```env
MONGO_URL=mongodb://localhost:27017/kinoman
JWT_SECRET=ваш-секретный-ключ
TELEGRAM_TOKEN=токен-вашего-telegram-бота
TELEGRAM_CHAT_ID=id-чата-или-канала
NODE_ENV=development
```

### Запуск

```bash
# Разработка (hot reload)
npm run dev

# Продакшен
npm run build
npm run prod
```

API будет доступен по адресу `http://localhost:4200/api`.
